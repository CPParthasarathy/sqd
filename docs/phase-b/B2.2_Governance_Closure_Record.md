# B2.2 Governance Closure Record

## Implemented controls

- The default branch is `main`.
- Changes to `main` require pull requests.
- Squash merging is the approved merge method.
- Merge commits and rebase merges are disabled.
- Linear history is required.
- Force pushes and branch deletion are restricted.
- Pull-request conversations must be resolved.
- Conventional Commits govern commit subjects.
- Semantic Versioning governs firmware releases.
- Release tags use annotated `vMAJOR.MINOR.PATCH` format.

## Bootstrap exception

Pull request #1 used a merge commit before the B2.2 controls were fully active. The published history is retained without force-pushing or rewriting.

All changes after governance activation must comply with the B2.2 policy.
